**Documento 1**  
**FICHA TÉCNICA DEL SOFTWARE**

**Propósito del documento:**

La ficha técnica constituye el núcleo descriptivo del software que se registrará ante Indecopi. Su función es identificar la obra de manera clara, completa y técnica, de modo que la autoridad pueda reconocerla como creación original y los equipos internos tengan un estándar uniforme de documentación.

Este documento debe redactarse con precisión, evitando ambigüedades y asegurando que cada campo refleje la naturaleza y funcionalidad del programa. A continuación, se presenta el formato ampliado con comentarios y ejemplos.

**1\. Título del Software**

**Redacción sugerida:**

“El presente programa de ordenador se denomina *CoffeeTrack ERP – Módulo Inventarios v1.0*. El título ha sido seleccionado para reflejar la naturaleza del sistema y su función principal dentro de la gestión empresarial. La denominación incluye el nombre de la suite principal (CoffeeTrack ERP), el módulo específico (Inventarios) y la versión (v1.0), lo que permite diferenciarlo de futuras actualizaciones o variantes.”

**Comentarios:**

* El título debe ser único y representativo.  
* Evitar nombres genéricos como “Sistema” o “Programa”.  
* Incluir versión ayuda a distinguir evoluciones del software.

**Ejemplo adicional:**

“SmartCafé POS – Control de Ventas v2.3” → refleja el producto, el módulo y la versión.

**2\. Lenguaje(s) de Programación Utilizado(s)**

**Redacción sugerida:**

“El software ha sido desarrollado principalmente en Python (para el backend y lógica de negocio), JavaScript con framework React (para la interfaz de usuario), y SQL en PostgreSQL (para la gestión de base de datos). Se han empleado librerías complementarias como Pandas para análisis de datos y Bootstrap para diseño responsivo. La elección de estos lenguajes responde a criterios de escalabilidad, compatibilidad multiplataforma y facilidad de mantenimiento.”

**Comentarios:**

* Se deben mencionar los lenguajes principales.  
* Las librerías se pueden incluir como referencia, pero no es obligatorio.  
* Justificar brevemente la elección de lenguajes aporta valor técnico.

**Ejemplo adicional:**

“Java (Spring Boot) para backend, Angular para frontend, y MongoDB como base de datos NoSQL.”

**3\. Funcionalidad Principal**

**Redacción sugerida:**

“El sistema tiene como funcionalidad principal la gestión integral de inventarios de café en tiempo real. Permite registrar entradas y salidas de productos, generar alertas automáticas cuando el stock alcanza niveles críticos, emitir reportes diarios y mensuales de ventas, y analizar la rotación de productos por punto de venta. Asimismo, se integra con módulos de facturación electrónica y control de proveedores, garantizando una administración eficiente de la cadena de suministro.”

**Comentarios:**

* La descripción debe ser breve pero técnica.  
* Se recomienda incluir las funciones clave y los beneficios.  
* Evitar frases vagas como “sirve para administrar cosas”.

**Ejemplo adicional:**

“Software para control de asistencia estudiantil, con registro biométrico, generación de reportes de puntualidad y exportación a Excel.”
