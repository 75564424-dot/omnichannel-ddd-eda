**Ejemplar del Software**

**Propósito del documento:**

El ejemplar del software constituye la evidencia técnica de la obra que se registrará. Su finalidad es demostrar la existencia, funcionalidad y originalidad del programa de ordenador. Este documento debe incluir el código fuente o ejecutable, un manual técnico que permita comprender su instalación y uso, y material gráfico que muestre la interfaz y los módulos principales.

**2\. Manual Técnico**

**Redacción sugerida:**

“El manual técnico tiene como objetivo guiar al usuario en la instalación, configuración y uso básico del software. Incluye los siguientes apartados:

**2.1 Requisitos del sistema**

* Sistema operativo: Windows 10 o superior / Linux Ubuntu 20.04.

* Lenguaje de programación: Python 3.10.

* Base de datos: PostgreSQL 14\.

* Navegador recomendado: Google Chrome o Mozilla Firefox.

**2.2 Instalación**

1. Descargar el archivo comprimido *CoffeeTrackERP\_v1.0.zip*.

2. Descomprimir en una carpeta local.

3. Instalar dependencias ejecutando el comando:

   **pip install \-r requirements.txt**

4. Configurar la base de datos ejecutando los scripts en la carpeta /db.

5. Iniciar el servidor con el comando:

   **python manage.py runserver**

**2.3 Configuración inicial**

* Crear un usuario administrador en la base de datos.

* Acceder al sistema mediante la dirección http://localhost:8000.

* Configurar parámetros de inventario inicial (productos, proveedores, puntos de venta).

**2.4 Uso básico**

* Ingreso al sistema mediante credenciales de usuario.

* Registro de entradas y salidas de productos.

* Generación de reportes de stock y ventas.

* Exportación de datos a Excel y PDF.

**2.5 Mantenimiento y actualización**

* Actualizar dependencias cada seis meses.

* Realizar copias de seguridad de la base de datos semanalmente.

* Documentar cambios en el archivo CHANGELOG.md.”

**Comentarios:**

* El manual debe ser claro y conciso, pero suficientemente detallado para demostrar la funcionalidad.

* Se recomienda incluir comandos, rutas y ejemplos de uso.

* No es necesario entregar manuales extensos, pero sí un documento que muestre que el software es operativo.
